# What's the game plan?
The goal of this mini-series is to build a mobile app by progressively integrating professional concepts. Here is our roadmap:
1. **Today:** Create the interface for a "Motivational Quotes" app (MotivApp) using [Thunkable](https://thunkable.com/), and connect it to a real, real-time Cloud database with [Firebase](https://firebase.google.com/).
2. **Part 2:** Make the app smart by replacing our database with calls to the Google Gemini API to generate unique quotes on the fly 🚀

# 15 minutes flat to prototype our app!

Our application will feature two screens: a home screen with an "Inspire me!" button, and a display screen that will randomly pull a quote stored in the Cloud.

## Step 1: Prepare the backend with Firebase
Firebase is Google's platform that provides ready-to-use backend services. We are going to use its **Realtime Database**.

1. Log into the [Firebase Console](https://console.firebase.google.com/) with a Google account and click on **Create a project** (name it *MotivApp*).
2. In the left menu, go to **Databases & Storage** > **Realtime Database** and click on **Create database**. Choose the test mode to start easily.
3. In the **Data** tab, we will structure our database with unique identifiers to make it robust. Click on the three little dots to the right of your database URL, choose **Import JSON**, and import a file containing this:

{% highlight json %}
{
  "quotes": {
    "quote_001": {
      "text": "Success is falling nine times and getting up ten.",
      "author": "Japanese Proverb"
    },
    "quote_002": {
      "text": "Make your life a dream, and a dream a reality.",
      "author": "Antoine de Saint-Exupéry"
    },
    "quote_003": {
      "text": "Innovation distinguishes between a leader and a follower.",
      "author": "Steve Jobs"
    }
  }
}
{% endhighlight %}

<img src="/content/images/firebase_rtdb_setup_en.jpg" alt="JSON structure in Firebase" style="display: block; margin: 0 auto;"><br/>

*Keep your database URL and your Web API key (available in the Firebase project settings) handy, we'll need them in Thunkable!*

The easiest way to find your API key is to:

1. Go to the Project overview page in the Firebase console.
2. Click on the button [+ Add app]
3. Click on the button </> to add a Web app.
4. Put *MotivApp* as app nickname, then click on [Register app].
5. You'll get a pop up with the values you need.

## Step 2: The Design (Visual Interface)

Create a free account on [Thunkable](https://x.thunkable.com/signup) and start a new project :
* On the main screen, you'll have a prompt box.
* At the bottom of that prompt box, find the line "Prefer to start from scratch?" then click on <u>Create Blank Project</u>".

<img src="/content/images/thunkable_pbox.jpg" alt="Thunkable Prompt Box" style="display: block; margin: 0 auto;"><br/>

* Put *MotivApp* as Project name, select *Health & Fitness*, make sure that *Use the Drag and Drop Builder* is selected.
* Click on [Create].

<img src="/content/images/thunkable_app_def.jpg" alt="Thunkable App definition" style="display: block; margin: 0 auto;"><br/>

**On Screen1 (Home):**
1. Drag a **Label** component to the center of the screen with the text "Welcome to MotivApp!". Center text and change font zsize using the options in the left tree view.
2. Drag a **Button** component below it. Change its text to "Inspire me!". Rename this component to `Btn_Inspire` in the left tree view.

<img src="/content/images/thunkable_screen_1.jpg" alt="Screen 1 of the MotivApp application" style="display: block; margin: 0 auto;"><br/>

**On Screen2 (Quote):**
1. Add a second screen (`Screen2`).
2. Drag two **Labels** one below the other. The first one (main text) will be renamed `Label_Quote`, the second one (smaller, for the author) will be `Label_Author`.
3. Drag a **Button** below with the text "Go Back" (`Btn_GoBack`).
4. **Connect your Realtime Database**
  * In your Thunkable tab, click the Settings gear icon in the sidebar.
  * Scroll to the Firebase section and paste the "apiKey" into the "Firebase API Key" field.
  * Still in the Firebase section, paste the "databaseURL" into the "Database URL" field.

<img src="/content/images/thunkable_screen_2.jpg" alt="Screen 1 of the MotivApp application" style="display: block; margin: 0 auto;"><br/>

## Step 3: The Logic (Blocks)
Switch from the **Design** tab to the **Blocks** tab. This is where the magic happens!

**Screen1 Logic:**
* Grab the `when Btn_Inspire Clicks do` block and drag the `Navigate to Screen2` block inside it.

**Screen2 Logic (Extracting JSON data):**
Here, we need to retrieve our dictionary of quotes, pick one at random, and then isolate its text and author.

1. Go to *Variables* and grab the `initialize cloud variable name` block. Name it `"quotes"` (this must *exactly* match the root node name of our JSON in Firebase so Thunkable syncs it automatically).
2. Grab the `when Screen2 Opens do` block.
3. Inside, we'll first list our keys. Create a temporary variable `random_key`. Assign to it the `random item of list` block. The list to provide will be generated by the `get object properties of` block (in the *Objects* category) applied to our `quotes` cloud variable.
4. Now that we have a random key (e.g., *quote_002*), let's create a `chosen_quote` variable. Assign to it the property (`get property of object` block) corresponding to `random_key` extracted from our `cloud quotes` object.
5. Finally, update your labels! Assign the `"text"` property of the `chosen_quote` object to `Label_Quote`, and its `"author"` property to `Label_Author`.
6. Grab the `when Btn_Inspire Clicks do` block and drag the `Navigate to Screen1` block inside it.

<img src="/content/images/thunkable_firebase_blocks_en.jpg" alt="Thunkable logic blocks with Firebase" style="display: block; margin: 0 auto;"><br/>

## Step 4: Test the beast!
No need to compile!
* Download the **Thunkable Live** app on your smartphone (iOS or Android).
* Log in with your account. Tap your project and test: you should see a new quote pulled directly from your Cloud database with every try 😉

## Conclusion

And there you have it! You've created a real mobile app connected to a professional backend. The huge advantage of this architecture is that you can now add, edit, or delete quotes directly from the Firebase console, and your users' app will update instantly, without them having to re-download it.
